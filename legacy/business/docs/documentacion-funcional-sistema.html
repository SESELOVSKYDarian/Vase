Documentacion Funcional del Sistema

Proyecto Teflon es una plataforma ecommerce multi-tenant con storefront publico, panel administrativo, gestion de catalogo, control comercial y capa de integracion con ERP o sistema de gestion.

Fecha de emision: 2026-03-20

1. Resumen ejecutivo

El sistema permite operar una tienda online con productos, categorias, marcas, precios segmentados, pedidos, contenido editable y sincronizacion externa de stock y precios. Fue pensado para cubrir la operacion comercial diaria sin depender de cambios manuales en codigo para tareas comunes del negocio.

Componentes principales

  • Frontend React + Vite
  • Backend Node.js + Express
  • Base PostgreSQL / Supabase
  • Admin integrado
  • API de integraciones

Objetivo operativo

  • Publicar la tienda
  • Administrar catalogo
  • Controlar precios y visibilidad
  • Gestionar pedidos
  • Sincronizar con sistema externo

2. Arquitectura general

2.1 Frontend

Incluye storefront y panel admin sobre una misma aplicacion.

Rutas publicas / /catalog /product/:id /cart /checkout /about

Cuenta /login /signup /profile /order-success /order-details

Admin /admin /admin/evolution /admin/legacy /admin/preview

2.2 Backend

La API organiza funcionalidades publicas, autenticadas, administrativas e integraciones.

Base Uso
/health Estado del servicio
/auth Login, registro, verificacion y sesion
/public Datos del storefront: tenant, catalogo, paginas, marcas y categorias
/checkout Validacion y creacion de pedido
/tenant Administracion de tienda, catalogo, precios y usuarios
/api/v1/integrations Sincronizacion con ERP / sistema de gestion

2.3 Base de datos

La base almacena tenants, usuarios, configuracion comercial, categorias, marcas, productos, pedidos, paginas, secciones y datos de integracion.

2.4 Topologia de despliegue

  • Frontend en Vercel
  • Backend en Render
  • Base de datos en Supabase

3. Modelo multi-tenant

Cada tenant representa una tienda con identidad propia. El sistema separa branding, configuracion, catalogo, paginas, usuarios e integraciones por tenant.

  • Resolucion por dominio o por X-Tenant-Id en entorno local
  • Configuracion visual independiente
  • Catalogo y reglas comerciales por tenant
  • Usuarios asociados al tenant

4. Funcionalidad del storefront

4.1 Home y contenido

  • Hero o slider principal
  • Bloques de productos destacados
  • Bloques de servicios
  • Paginas editables como home y about

4.2 Catalogo

  • Buscador
  • Listado paginado
  • Filtro por categoria
  • Filtro por marca
  • Filtro por precio minimo y maximo
  • Filtro solo con stock
  • Orden por nombre, precio y stock
  • Version responsive para mobile
  • Menu y panel de filtros adaptados a pantallas chicas
Las categorias raiz pueden plegar y desplegar sus hijas. El catalogo tambien soporta productos agrupados por variaciones para mostrar un producto raiz y expandir sus variantes.

4.3 Productos destacados

La home puede mostrar productos destacados definidos desde admin. Estas tarjetas usan el detalle corto del producto para una comunicacion mas comercial y rapida.

4.4 Ficha de producto

  • Imagenes
  • Nombre y marca
  • Precio segun reglas de visibilidad
  • Detalle corto
  • Descripcion larga
  • Stock
  • Productos relacionados
  • Especificaciones tecnicas

4.5 Cuenta y pedidos

  • Login y registro
  • Perfil
  • Historial de pedidos
  • Detalle de pedido

5. Carrito y checkout

  1. El usuario agrega productos al carrito.
  2. Revisa cantidades y resumen.
  3. Completa los datos del pedido.
  4. El backend valida disponibilidad y datos comerciales.
  5. Se genera el pedido y se deriva al flujo correspondiente.

Modos contemplados actualmente:

  • Transferencia bancaria
  • Retiro / efectivo
  • Estructura preparada para medios online futuros

6.7 Pedidos

El admin puede consultar pedidos para seguimiento operativo y comercial.

7. Integracion con ERP o sistema de gestion

El backend expone endpoints de integracion para sincronizar productos, stock y precios. La integracion acepta token tradicional y tambien esquemas compatibles con consumer key / consumer secret.

  • Ping de conectividad
  • Sincronizacion por external_id
  • Actualizacion de stock y precios
  • Alta o actualizacion de productos
  • Compatibilidad con payloads de sistemas de gestion
Controla el sistema externo Controla el ecommerce
Stock, precio base, estado base, codigo externo Visibilidad web, contenido editorial y bloqueo manual

8. Flujos principales

8.1 Compra

  1. El visitante entra a la tienda.
  2. Navega por categorias, marcas o catalogo completo.
  3. Filtra, expande variaciones o entra a una ficha.
  4. Agrega al carrito.
  5. Completa checkout.
  6. Se genera el pedido.

8.2 Administracion

  1. El admin inicia sesion.
  2. Ingresa al panel.
  3. Actualiza contenido, catalogo, precios o apariencia.
  4. Guarda y publica cambios.

8.3 Integracion

  1. El sistema externo arma un payload JSON.
  2. Consume el endpoint de sincronizacion.
  3. El backend crea o actualiza productos.
  4. La tienda refleja stock y precios sincronizados.

9. Diferenciales

  • Multi-tenant real
  • Storefront y admin en la misma plataforma
  • Control de visibilidad de precios
  • Variaciones con producto raiz
  • Especificaciones tecnicas administrables
  • Integracion ERP lista para uso
  • Contenido editable sin tocar codigo

10. Consideraciones operativas

  • La configuracion CORS debe coincidir con la URL publica del frontend.
  • Las integraciones deben enviar external_id estable.
  • El manejo de archivos locales conviene migrarlo a storage externo para una operacion mas robusta.
  • La calidad del catalogo depende de la consistencia entre productos, marcas y categorias.

11. Conclusion

Proyecto Teflon cubre el nucleo operativo de un ecommerce administrable: tienda publica, contenido, catalogo, precios, pedidos, usuarios e integracion externa. No es solo una vidriera, sino una plataforma preparada para operar comercialmente y crecer sobre una arquitectura separada entre frontend, backend y base de datos.
